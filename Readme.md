# AI Journal Analysis System

This project is an AI-assisted journal system where users can write journal entries after immersive nature sessions (forest, mountain, ocean, etc.) and receive emotional insights generated by an LLM.

The system stores journal entries, analyzes emotions using Google Gemini, and generates insights about the user's mental state over time.

---

# Features

- Write and store journal entries
- AI emotion analysis using Gemini
- Extract important keywords from journals
- Generate a short emotional summary
- View previous journal entries
- Get insights about emotional patterns

## Setup Instructions

### 1. Clone the repository

```bash
git clone <https://github.com/AnkuThakur704/AI-Journal>
cd ai-journal
```
### 2. Install backend dependencies
```bash
cd backend
npm install
```
### 3. Install frontend dependencies
```bash
cd ../frontend
npm install
```
### 4. Environment Variables
```bash
Create a .env file inside the backend folder.

PORT=8080
GEMINI_API_KEY=your_api_key
MONGO_URI=your_mongodb_connection_string
```
### 5. Run Backend
```bash
cd backend
npm start
```
### Server runs at:
```bash
http://localhost:8080
```
### 6. Run Frontend
```bash
cd frontend
npm run dev
```
## API Endpoints
### Analyze Journal

POST /api/journal/analyze
```bash
Request
{
  "text": "I felt calm today after listening to the rain."
}
```
```bash
Response
{
  "success": true,
  "result": {
    "emotion": "calm",
    "keywords": ["rain", "peace", "nature"],
    "summary": "User experienced calm and relaxation."
  }
}
```
### Save Journal Entry

POST /api/journal
```bash
Request
{
  "userId": "123",
  "ambience": "forest",
  "text": "I felt calm today after listening to the rain.",
  "result": {
    "emotion": "calm",
    "keywords": ["rain", "peace", "nature"],
    "summary": "User experienced calm."
  }
}
```
### Get Journal Entries

GET /api/journal/:userId
```bash
Returns all journal entries for a user.
```

### Get Insights

GET /api/journal/insights/:userId
```bash
Returns:

totalEntries

topEmotion

mostUsedAmbience

recentKeywords
```

## Future Improvements

The current system provides the core functionality for journal analysis and insights.  
Several enhancements can be implemented to improve scalability, security, and user experience.

### Planned Enhancements

- **User Authentication**  
  Add secure login and account management to associate journal entries with authenticated users.

- **Emotion Trend Visualization**  
  Provide visual insights (charts/graphs) to help users track emotional patterns over time.

- **Caching LLM Analysis Results**  
  Cache previously analyzed journal entries to reduce LLM calls, lower costs, and improve response time.

- **Rate Limiting for API Protection**  
  Implement request limits to prevent abuse and protect the LLM API from excessive usage.

- **Background Processing for Analysis**  
  Move LLM analysis to background jobs using queues (e.g. Redis) to improve scalability and responsiveness.
